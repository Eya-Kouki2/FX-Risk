Guide du Débutant : La Plateforme FX Risk

Bonjour ! Ce document est écrit pour vous, même si vous n'avez aucune connaissance en informatique. Nous allons utiliser des mots simples et des comparaisons avec la vie de tous les jours pour vous expliquer ce qu'est ce projet, comment il fonctionne et pourquoi on l'utilise.

1. Qu'est-ce que ce projet (FX Risk Platform) ?

Imaginez une grande banque qui gère l'argent de ses clients. Parfois, ces clients veulent échanger de l'argent d'un pays contre de l'argent d'un autre pays (par exemple, échanger des Euros contre des Dollars). C'est ce qu'on appelle une opération de Change (ou FX pour Foreign Exchange en anglais).

Ces opérations impliquent parfois des millions d'euros. Si quelqu'un fait une erreur de frappe, ou si un client n'a pas les fonds nécessaires, la banque peut perdre beaucoup d'argent. C'est un Risque.

Notre projet, la FX Risk Platform (Plateforme de Risque FX), est comme un super-vigile intelligent pour la banque. C'est un site web sécurisé qui surveille toutes ces transactions en direct, analyse si elles sont dangereuses ou non, et avertit les bonnes personnes en cas de problème.

2. Qui utilise ce programme ? (Les rôles)

Dans une banque, tout le monde ne fait pas la même chose. Notre plateforme sépare les utilisateurs en 5 grandes équipes :

  • Le Front Office (Les Vendeurs) : Ce sont ceux qui parlent aux clients et enregistrent les demandes d'échange d'argent. (Exemple : Le caissier du supermarché qui scanne les articles).
  • Le Back Office (Les Vérificateurs) : Ils s'assurent que les informations entrées par le Front Office sont correctes avant d'envoyer l'argent. (Exemple : Le vigile qui vérifie votre ticket de caisse à la sortie).
  • Le Middle Office (Risk Team) : Ce sont les experts qui surveillent la plateforme toute la journée pour voir s'il y a des "Alertes Rouges" (comme une transaction anormalement grosse). Ils peuvent bloquer une opération.
  • Le Manager : Le chef qui regarde les statistiques générales pour voir si tout se passe bien (il a accès à un grand tableau de bord).
  • L'Administrateur : Le technicien qui s'occupe de donner les accès (mots de passe) aux autres employés.

3. C'est quoi "npm run dev" ? Pourquoi doit-on taper ça ?

Si vous avez déjà ouvert ce projet, on vous a sûrement dit de taper la commande suivante dans une fenêtre noire (le terminal) :

npm run dev
L'analogie du restaurant :
Imaginez que ce projet informatique est un restaurant.
- Les fichiers du code que vous avez sur votre ordinateur sont la cuisine, les tables et les ingrédients.
- Mais un restaurant ne sert à rien si on n'allume pas les lumières, les fours et si on n'ouvre pas la porte d'entrée !

La commande npm run dev signifie en langage humain : "S'il te plaît, allume le moteur du site web pour que je puisse l'utiliser sur mon ordinateur."

Voici l'explication mot à mot :

  • npm : C'est le nom du robot à tout faire de notre ordinateur (Node Package Manager). C'est lui qui écoute nos ordres.
  • run : C'est le mot anglais pour "Lancer" ou "Faire tourner".
  • dev : C'est l'abréviation de "Développement". Cela dit au robot d'allumer le site web en "mode privé/atelier", ce qui vous permet de voir le site sur votre propre écran sans qu'il ne soit visible par tout le monde sur internet.

Une fois que vous avez tapé ça et appuyé sur "Entrée", l'ordinateur travaille quelques secondes, puis il vous donne une adresse web qui ressemble à http://localhost:5173.

Localhost veut dire "Mon propre ordinateur". C'est comme si le site web était diffusé uniquement sur votre téléviseur privé. Vous pouvez alors ouvrir Google Chrome, taper cette adresse en haut, et le site apparaît !

4. Comment le site se souvient-il des informations ? (Supabase)

Si vous ajoutez une nouvelle transaction sur le site, comment fait-il pour s'en souvenir le lendemain ? Il utilise une Base de données.

Dans ce projet, nous utilisons un outil qui s'appelle Supabase.

Imaginez Supabase comme une immense armoire à tiroirs (un classeur) ultra-sécurisée qui se trouve dans un bâtiment lointain (le cloud). Quand vous tapez une nouvelle opération sur votre écran, votre ordinateur envoie instantanément une petite lettre à Supabase qui range l'information dans le bon tiroir. Le lendemain, quand vous vous connectez, Supabase vous renvoie le contenu du tiroir pour l'afficher sur votre écran.

5. De quoi est fabriqué ce projet ? (Les Outils)

Même si vous n'avez pas besoin de les modifier, il est intéressant de savoir quels "matériaux" ont été utilisés pour construire cette plateforme. Voici les principaux :

  • React : C'est comme les briques LEGO du site. Au lieu de dessiner toute la page d'un coup, on crée des petits blocs (un bouton, un tableau, un menu) qu'on assemble ensuite. Cela permet de construire des choses très complexes mais solides.
  • Tailwind CSS : C'est la boîte de peinture et de décoration. C'est grâce à Tailwind que le site a de belles couleurs, que les boutons sont arrondis et que le texte est bien aligné.
  • TypeScript : C'est le correcteur d'orthographe strict pour les développeurs. Il vérifie que le code n'a pas d'erreurs avant même qu'on allume le site, ce qui évite beaucoup de pannes inattendues.
  • Vite : C'est le moteur ultra-rapide. C'est lui qui fait le lien entre tous les fichiers de code et votre navigateur internet pour que le site s'affiche en une fraction de seconde quand vous tapez npm run dev.
  • Supabase : Comme vu plus haut, c'est l'immense armoire forte (la base de données) qui stocke vos informations (utilisateurs, alertes, montants d'argent) en toute sécurité sur internet.

Résumé

Pour faire simple, ce que vous avez sous les yeux, c'est :

  1. Un outil métier (FX Risk Platform) pour surveiller et sécuriser de gros échanges d'argent dans une banque.
  2. Une clé de contact (npm run dev) qui permet de démarrer le moteur de cet outil sur votre ordinateur.
  3. Des comptes de démonstration (comme front@fxrisk.demo) pour que vous puissiez tester l'outil en vous mettant dans la peau des différents employés de la banque.
Et voilà ! Vous savez maintenant l'essentiel sur ce projet. Il a été conçu pour être très facile à utiliser à l'écran, même si sa mécanique interne (le code) est très sophistiquée.
